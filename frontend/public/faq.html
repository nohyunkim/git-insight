자주 묻는 질문

비공개 저장소 활동도 보이나요?

아닙니다. Git Insight는 GitHub 공개 API를 기반으로 동작하므로 비공개 저장소, 조직 내부 작업, 회사 계정 협업 기록은 충분히 반영되지 않을 수 있습니다. 실제 역량 전체를 보여주려면 공개 저장소 설명과 README를 함께 정리하는 편이 좋습니다.

기간을 바꾸면 결과가 달라지는 이유는 무엇인가요?

7일, 30일, 90일, 6개월, 1년은 서로 다른 집계 창입니다. 기간이 길어질수록 더 넓은 공개 활동이 포함되므로 이벤트 수, 활동 일수, 언어 분포, AI 요약 문구가 함께 달라질 수 있습니다.

AI 피드백이 README를 읽고 평가하나요?

아닙니다. 현재 AI 피드백은 공개 활동 통계를 바탕으로 문장을 생성합니다. 저장소의 실제 README 본문이나 코드 내용을 직접 읽는 구조는 아니므로, README를 이미 잘 정리했더라도 보수적인 추천이 다시 나올 수 있습니다.

왜 같은 계정을 다시 검색했는데 문구가 조금 달라질 수 있나요?

공개 GitHub 데이터는 계속 갱신되고, 서비스는 일정 시간 동안 캐시를 사용합니다. 또한 AI 요약은 규칙 기반 요약을 바탕으로 생성되므로, 시점과 데이터 변화에 따라 세부 표현이 조금 달라질 수 있습니다.

왜 최근 1년 결과는 생각보다 단순하게 보일 수 있나요?

GitHub 공개 Events API는 오래된 이벤트를 모두 안정적으로 제공하지 않을 수 있습니다. 그래서 기간이 길어질수록 최근 구간보다 세부 이벤트 분포가 단순하게 보이거나 일부 기록이 덜 잡힐 수 있습니다. 이런 경우에는 30일, 90일 결과도 함께 비교해서 보는 편이 좋습니다.

저장한 결과 비교는 어떤 기준으로 이뤄지나요?

저장 기록 비교는 같은 GitHub 사용자, 같은 분석 기간의 저장본끼리 현재 기록과 이전 기록을 나란히 비교하는 방식입니다. 이벤트 수, Push 수, 활동 일수, PR/이슈 기록, 공개 저장소 수 같은 공개 지표의 변화를 바탕으로 성장 포인트와 보완 포인트를 요약합니다.

언어 분포가 실제로 제가 쓰는 기술과 다르게 보일 수 있나요?

그럴 수 있습니다. 언어 분포는 공개 저장소 기준으로 계산되므로 비공개 프로젝트, 회사 저장소, 오래된 공개 저장소의 비중에 따라 결과가 달라질 수 있습니다. 실제 주력 기술을 더 잘 보여주고 싶다면 대표 프로젝트 README와 설명 문구를 함께 정리하는 편이 좋습니다.

검색 결과가 적게 나오거나 빈약해 보일 때는 어떻게 해야 하나요?

먼저 공개 저장소와 최근 공개 활동이 실제로 있는지 확인해 보세요. 이후 README 보강, 작은 단위의 커밋 기록, PR 설명, 이슈 메모처럼 공개 기록의 맥락을 늘리면 결과 화면도 더 풍부하게 읽히는 경우가 많습니다.

광고가 들어오면 결과 화면이 바뀌나요?

광고를 도입하더라도 검색 버튼, 결과 액션 버튼, 탐색 요소와 혼동되지 않도록 배치할 계획입니다. 콘텐츠를 가리거나 클릭을 유도하는 형태의 배치는 지양하며, 정책과 사용자 경험을 함께 고려합니다.

문의나 오류 제보는 어디로 하면 되나요?

하단의 문의/오류 제보 링크를 통해 접수할 수 있습니다. 재현 상황, GitHub 아이디, 기간, 스크린샷을 함께 보내주면 확인에 도움이 됩니다.
