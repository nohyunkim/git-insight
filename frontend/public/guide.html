Git Insight 지표 해석 가이드

이 문서는 Git Insight 결과를 단순 숫자가 아니라 활동 패턴으로 읽을 수 있도록 만든 해석 가이드입니다.

1. 이벤트 수

이벤트 수는 선택한 기간 동안의 공개 활동 총량입니다. 큰 변동 없이 꾸준히 유지된다면 루틴형 활동, 특정 기간 급증한다면 릴리즈/스프린트 집중 구간일 가능성이 큽니다.

2. Push 비율

Push 비율은 전체 이벤트 중 코드 반영 활동의 비중을 의미합니다. 비율이 높으면 구현 중심, 이슈/PR 관련 이벤트 비중이 높으면 협업/리뷰 중심 활동일 가능성이 있습니다.

3. 활동 일수

활동 일수는 기간 내 실제로 활동이 발생한 날짜 수입니다. 같은 이벤트 수라도 활동 일수가 높으면 분산형, 낮으면 특정 기간 집중형으로 해석할 수 있습니다.

4. 언어 분포

언어 분포는 공개 레포지토리 기준이므로 현재 집중 스택 파악에는 유용하지만, private 작업이나 실험 저장소는 반영되지 않을 수 있습니다.

주의사항

분석은 공개 데이터 기준이며, 계정 공개 설정 및 API 응답 상태에 따라 일부 지표가 달라질 수 있습니다. 결과는 참고 지표로 활용하고, 중요한 판단 전에는 원본 활동 내역을 함께 확인하는 것을 권장합니다.
