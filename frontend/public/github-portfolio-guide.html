좋은 GitHub 포트폴리오 만드는 법

GitHub 포트폴리오는 저장소 수보다 구조가 더 중요합니다. 처음 보는 사람이 핵심 프로젝트와 작업 흐름을 빠르게 이해할 수 있어야 설득력이 생깁니다.

핵심만 보기
  • 대표 프로젝트 몇 개를 먼저 정리하는 편이 더 읽기 좋습니다.
  • README는 결과보다 맥락과 선택 이유를 설명해야 합니다.
  • 저장소 흐름과 최근 활동도 포트폴리오 인상에 영향을 줍니다.

대표 프로젝트를 먼저 고르세요

모든 저장소를 같은 비중으로 보여 주면 핵심이 흐려질 수 있습니다. 보여 주고 싶은 프로젝트 몇 개를 먼저 정리하고, 나머지는 보조 자료처럼 두는 편이 더 깔끔합니다.

README는 맥락을 설명해야 합니다

무엇을 만들었는지만 적는 것보다 왜 만들었는지, 어떤 문제를 해결하려 했는지까지 함께 적는 문서가 훨씬 설득력 있습니다. 개인 프로젝트일수록 이 차이가 크게 보입니다.

활동 흐름도 함께 보입니다

GitHub는 결과물뿐 아니라 작업 습관도 드러나는 공간입니다. 오랫동안 방치된 대표 저장소는 README를 다듬거나 간단한 개선 기록을 남겨 흐름을 다시 살려 두는 것도 도움이 됩니다.

설명 가능한 저장소가 더 강합니다

프로젝트 목적, 주요 기능, 기술 선택 이유, 실행 방법을 스스로 설명할 수 있어야 저장소가 더 설득력 있게 읽힙니다. 정돈된 몇 개의 저장소가 산만한 많은 저장소보다 더 좋은 인상을 줍니다.
