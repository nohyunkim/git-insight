좋은 GitHub 포트폴리오를 만드는 법

GitHub 포트폴리오는 저장소 수보다 읽히는 방식이 더 중요합니다. 외부에서 봤을 때 무엇을 만들었는지, 왜 만들었는지, 어느 정도까지 직접 다뤘는지를 빠르게 이해할 수 있어야 합니다.

보여주고 싶은 프로젝트를 먼저 정리하세요

모든 저장소가 포트폴리오 역할을 해야 하는 것은 아닙니다. 핵심 프로젝트 몇 개를 명확하게 정리해 두는 편이 훨씬 읽기 좋습니다.

README는 결과보다 맥락을 전달해야 합니다

왜 이 프로젝트를 만들었는지, 어떤 문제를 해결하려 했는지, 어떤 구조로 설계했는지를 README에서 설명해야 프로젝트의 가치를 빠르게 이해할 수 있습니다.
